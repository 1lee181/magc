Website Design Principles

Analysis

An analysis of real world websites evaluated against core UX design principles we learned in class, highlighting what works and what doesn't. This is from the CS 1XD3 Lab 3.2: Design Principles on 21 January 2026. Were following the design principles laid out in Design Principles textbook chapter link

bad website (dundas valley hobby) (link)
  • Consistency
    • Link inconsistency: In the Products list, only the text within each bullet is clickable rather than the entire list item, which is inconsistent with standard linked list behavior.
    • Header inconsistency: The Products page has a page-level header, but this pattern is not followed on other pages. The Service page lacks a "Service" header and jumps directly to a subsection ("Gift Cards").
    • Text styling inconsistency: The Contact page uses bold text (e.g., "Attention") that is not used or styled consistently elsewhere.
    • Accessibility inconsistency: Images displayed together do not consistently include alt attributes; some images lack alternative text entirely.
  • Ease of Navigation
    • The links and navigation sections are not consistent. When clicking on the products page, the colour and font of the navigation boxes on the left are different than the nav bar at the top.
  • Universal Usability and Accessibility
    • Missing Alt attributes for images: Several images on this webpage lack descriptive alt text leading to users using a screen reader to not be able to interpret image content.
    • Poor Responsive Web Design: As you transition from device to device, the layout stays the same, causing inconsistent layouts across devices. No media queries are used to help organize content for mobile users. This webpage fails at universal usability and responsiveness as it was clearly designed for desktop users only.
    • Buttons for mobile users: An inherent issue with this webpage is the lack of accommodations for mobile users. Anything smaller than a laptop screen, the buttons progressively begin to shrink making it difficult to interact with.
    • Screen management for mobile users: As the screen progressively gets smaller, the content is shrunk, while blank space is used to fill the remainder of the page. It makes the page less readable and interactive.
  • Error Prevention
    • A significant amount of errors presented to the user within the HTML. Once checked with the W3C validation service, 14 errors and 18 warnings were found. In the CSS there is also a parse error.
Good Website (McMaster Deltahacks Hackathon Website) (link)
  • Consistency
    • Consistent visual language: Illustrations, colour palette, icon style, and rounded shapes are used uniformly across all sections, creating a cohesive visual identity.
    • Consistent section hierarchy: Each section (About, Speakers, Sponsors, FAQ) follows a clear pattern: section header → grouped content → uniform spacing, making scanning easy.
    • Consistent interaction cues: Clickable elements (links, cards, icons) behave and appear consistently throughout the site, reducing guesswork.
  • Ease of Navigation
    • Navbar stays consistently at the top of the screen when scrolling through the website. With easy access to the homepage.
    • Navbar has good contrast with the background and the rest of the site.
    • All links work properly and are intuitive.
  • Error Prevention
    • Very mobile and device-friendly. Webpage/design gets updated depending on the size of a device, increasing usability and error prevention.
